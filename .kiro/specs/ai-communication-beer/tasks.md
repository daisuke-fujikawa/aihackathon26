# Implementation Plan

- [x] 1. プロジェクトセットアップと依存関係追加
  - canvas-confetti パッケージを追加する
  - OpenAI TTS API 用の環境変数 `OPENAI_API_KEY` を `.env.local` に追加する
  - サンプルの posts テーブルスキーマを削除し、今回のフィーチャーで不要な既存コードを整理する
  - ヨイさん画像アセットの配置用ディレクトリ（`public/yoi/`, `public/bg/`）を作成し、プレースホルダー画像を配置する
  - _Requirements: 11.1, 11.2_

- [x] 2. セッション状態管理の構築
  - セッション全体のフェーズ（セットアップ→待機→録音中→処理中→発話中→乾杯）を管理するReact Contextプロバイダーを実装する
  - 参加者リスト、メッセージ履歴、乾杯回数、酔度レベル、現在のヨイさん画像キーをステートとして管理する
  - 酔度レベルの自動更新ロジック（経過時間に基づきレベル1→2→3）を実装する
  - ヨイさんの画像キー切り替えロジック（酔度レベルベース＋トリガー一時切り替え）を実装する
  - ファシリテーション設定（沈黙閾値、パス間隔、休憩間隔、乾杯閾値）のデフォルト値を定義する
  - _Requirements: 1.1, 2.2, 8.3, 10.4_

- [x] 3. 音声認識フックの実装
- [x] 3.1 (P) Web Speech API を使った音声認識のカスタムフックを作成する
  - `webkitSpeechRecognition` を使って日本語のリアルタイム音声認識を行う
  - `continuous: true`, `interimResults: true` で中間結果と最終結果を取得する
  - セッションタイムアウト時の自動再接続ロジック（100-300ms遅延で `onend` から再開）を実装する
  - React 19 Strict Mode での二重起動防止（ref フラグ）を行う
  - マイク権限拒否時のエラーハンドリングを実装する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

- [x] 3.2 (P) マイク音量レベルを計測するカスタムフックを作成する
  - `AudioContext` + `AnalyserNode` を使ってリアルタイム音量を取得する
  - `requestAnimationFrame` ループで音量データを読み取り、正規化した音量値（0.0-1.0）を返す
  - 音量閾値ベースの発話中判定と累積発話時間の計測を行う
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 4. AI応答生成APIの実装
- [x] 4.1 (P) ヨイさんキャラクターでAI応答を生成するAPIルートを作成する
  - 既存の Anthropic クライアント（`src/lib/ai.ts`）を使って Claude API を呼び出す
  - ヨイさんのキャラクター定義（一人称「わし」、語尾の特徴、酔っ払い口調）をシステムプロンプトとして設定する
  - トリガー種別（沈黙キラー、パス出し、帰宅リマインド、休憩提案）に応じたプロンプト調整を行う
  - 参加者名リストと直近の会話履歴をコンテキストとして渡す
  - 応答の長さを制限（max_tokens: 200）し、音声合成の遅延を最小化する
  - APIエラー時のレスポンスハンドリングを実装する
  - _Requirements: 8.1, 8.2, 8.4_

- [x] 4.2 (P) テキストを音声に変換するTTS APIルートを作成する
  - OpenAI TTS API（`POST /v1/audio/speech`）を fetch で直接呼び出す
  - モデル `tts-1`、ボイス `onyx`、フォーマット `mp3`、速度 0.8 をデフォルトとする
  - 音声バイナリをそのままレスポンスとして返す
  - APIエラー時のハンドリングを実装する
  - _Requirements: 9.1, 9.2_

- [x] 5. 音声再生フックの実装
  - Web Audio API でTTSから受け取った音声バッファを再生する
  - `AudioBufferSourceNode` の `playbackRate` を 0.95-1.05 のランダム値に設定し、酔っ払い風の揺らぎを加える
  - `BiquadFilter`（lowshelf）を通して音質を調整する
  - 再生中・完了の状態を管理し、再生完了時にコールバックを発火する
  - _Requirements: 9.1, 9.2, 9.3, 9.4_

- [x] 6. キーワード検出の実装
  - 音声認識の中間結果をリアルタイムで監視し、キーワードマッチングを行う
  - 「乾杯」「かんぱい」検出時に即座に乾杯イベントを発火する
  - 「終電」「明日」「何時」「駅」等の帰宅関連キーワード検出時に帰宅リマインドイベントを発火する
  - 検出後のクールダウン（5秒）で重複発火を防止する
  - _Requirements: 5.1, 5.2, 5.3, 6.1, 6.2_

- [x] 7. ファシリテーションエンジンの実装
  - タイマーベースで全体の沈黙時間（10秒閾値）を監視し、沈黙キラーを発動する
  - 一定時間（30秒）ごとに参加者リストからランダムに未指名者を選びパス出しを行う
  - 経過時間（30分）または乾杯回数（3回）到達時に休憩を提案する
  - 会話の区切り（沈黙後の発話開始）を検知し、休憩提案の好機を判断する
  - 帰宅キーワード検出イベントを受け取り、帰宅リマインドのAI応答生成をトリガーする
  - AI応答生成中は新たなトリガーを抑制する
  - 各トリガー発動時にヨイさんの画像キーを専用画像に切り替える（pass, clock, restroom）
  - _Requirements: 3.1, 3.2, 3.3, 4.1, 4.2, 4.3, 6.1, 6.2, 7.1, 7.2_

- [x] 8. UI コンポーネントの実装
- [x] 8.1 参加者名入力画面を作成する
  - 飲み会参加者の名前を入力するフォームを実装する
  - 名前リストを確定してセッションを開始する機能を実装する
  - ダークモードUIで居酒屋の暗い環境でも見やすいデザインにする
  - _Requirements: 3.2, 10.1_

- [x] 8.2 (P) チャットパネルとメッセージ吹き出しを作成する
  - メッセージ履歴を時系列で表示するチャットエリアを実装する
  - ヨイさんのメッセージにはアバター画像（現在のヨイさん画像キーに対応）を左側に表示し、酔っ払い風スタイル（吹き出し微回転CSS）を適用する
  - ユーザーのメッセージは右寄せで表示する
  - 新しいメッセージ追加時に自動スクロールする
  - 背景画像（`public/bg/background.png`）を全画面に表示する
  - _Requirements: 10.1, 10.2, 10.3, 10.4_

- [x] 8.3 (P) ビールジョッキビジュアライザーを作成する
  - 音量レベルと累積発話時間に基づいて泡の量・液面の高さが変動するビールジョッキのビジュアルを実装する
  - CSSアニメーションで発話量変化を0.5秒以内にリアルタイム反映する
  - _Requirements: 2.1, 2.2, 2.3_

- [x] 8.4 (P) マイク状態インジケーターを作成する
  - セッションフェーズに応じて4状態（停止中・録音中・処理中・発話中）をアイコンで切り替え表示する
  - マイクの開始・停止ボタンとしても機能させる
  - _Requirements: 10.5_

- [x] 8.5 (P) 乾杯クラッカー演出を実装する
  - 乾杯イベント発火時に `canvas-confetti` を使って画面全体にクラッカー演出を表示する
  - 色はビール色系（ゴールド、アンバー、白泡）にカスタマイズする
  - 乾杯検出から演出開始までを1秒以内にする
  - _Requirements: 5.1, 5.3_

- [x] 9. メインインタラクションループの統合
  - 音声認識→キーワード検出→ファシリテーションエンジン→AI応答生成→TTS→音声再生→音声認識再開の一連のフローを結合する
  - セッションフェーズの遷移（LISTENING→PROCESSING→SPEAKING→LISTENING）を正しく制御する
  - 乾杯検出時の並行フロー（KANPAI→クラッカー＋歓声→LISTENING復帰）を実装する
  - AI応答テキストの生成中インジケーター表示を連動させる
  - 音声認識からAI応答の音声再生開始までの3秒以内のレスポンス目標を検証する
  - 全コンポーネントをページに配置し、レイアウトを完成させる
  - _Requirements: 1.5, 5.2, 5.3, 8.3, 8.5, 9.3, 9.4_

- [x] 10. エラーハンドリングとポリッシュ
  - マイク権限拒否時の案内メッセージを表示する
  - Chrome以外のブラウザでアクセスした場合の案内メッセージを表示する
  - Claude API / OpenAI TTS API失敗時の自動リトライ（3回、1秒間隔）を実装する
  - TTS API失敗時のフォールバック（テキストのみ表示）を実装する
  - Web Speech API セッション切断時のサイレント再接続が正常動作することを確認する
  - _Requirements: 1.4, 8.4, 11.3, 11.4_
