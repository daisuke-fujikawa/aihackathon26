- [x] 1. 設定値を FacilitationConfig に集約する

  - 既存 `SessionStateProvider` の初期化とテストが新フィールド追加後も成立することを確認する
  - _Requirements: 4.1, 4.2_

- [x] 2. 応答長ガードを実現する
- [x] 2.1 (P) 応答切り詰めユーティリティを新設する
  - 日本語の自然な文末（句点・感嘆符・疑問符・波ダッシュ）を優先して切り詰める純粋関数を追加する
  - サロゲートペア（絵文字）を正しく数え、閾値以内なら原文、閾値超過時は末尾を整える
  - 句読点が無い場合は末尾に「…」を付与して文字数上限を必ず満たす
  - 切り詰めた場合に呼び出し側が検知できる戻り値を返す
  - _Requirements: 1.1, 1.4, 4.4_

- [x] 2.2 (P) 応答切り詰めユーティリティのユニットテストを追加する
  - 閾値内ならそのまま返ること、句点で自然に切れること、句読点なしで「…」が付くこと、絵文字混在で文字数破綻しないことを検証する
  - _Requirements: 1.1, 1.4_

- [x] 2.3 /api/chat のシステムプロンプトと生成パラメータを調整する
  - ヨイさんのキャラクターは維持したまま、1文・40字以内・前置き相槌禁止・箇条書き禁止を強調する指示文に差し替える
  - `max_tokens` を短文応答に必要十分な上限（80）に引き下げる
  - `truncated` を含む切り詰め結果を後処理で適用し、応答として返す
  - 生成時・切り詰め発生時の理由と文字数をサーバーログに出力する
  - _Requirements: 1.1, 1.2, 1.3, 1.5, 4.3, 4.4_

- [x] 2.4 /api/chat の既存テストを新仕様に合わせて更新する
  - Anthropic クライアントモックに渡される `max_tokens` とシステムプロンプト更新を検証する
  - 長文応答が切り詰められた場合にレスポンス本文が閾値以内に収まることを検証する
  - _Requirements: 1.1, 1.3, 1.4_

- [x] 3. 発話ゲートとクールダウンをクライアント側に実装する

  - 既存の TTS 失敗時フォールバック（テキストのみ表示）を壊さない
